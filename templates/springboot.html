Spring Boot Overview

Spring Boot is an open-source Java-based framework used to create standalone, production-grade Spring-based Applications. It aims to make it easy to create Spring-powered, production-grade applications and services with minimum fuss. It is widely used for building web applications, microservices, and APIs.

Key Features

  1. Auto-configuration: Spring Boot automatically configures your application based on the JAR dependencies you have added to the project.
  2. Standalone: Spring Boot applications are standalone and do not require a separate application server, as they embed an embedded server (like Tomcat, Jetty, or Undertow).
  3. Convention over Configuration: Spring Boot follows the principle of "convention over configuration," providing sensible defaults and reducing the need for manual configuration.
  4. Spring Boot Starter: Provides a set of starter dependencies that simplify the inclusion of common dependencies required for various types of applications.
  5. Spring Boot Actuator: Provides production-ready features to help you monitor and manage your application, including metrics, health checks, and more.
  6. Spring Boot DevTools: Provides tools to improve the development experience, including automatic application restarts, live reload of changes, and more.
  7. Embedded Server Support: Allows embedding servlet containers like Tomcat, Jetty, or Undertow directly into your application.
  8. Spring Boot CLI: Command Line Interface (CLI) allows you to quickly prototype and develop Spring Boot applications using a command-line interface.
  9. Spring Boot Security: Provides comprehensive security features, including authentication, authorization, and protection against common security vulnerabilities.

Overall, Spring Boot simplifies the process of building and deploying Spring applications, making it a popular choice for Java developers.

Spring Boot Project Overview

A Spring Boot project follows a structured approach to organize its components and configuration. Here's an overview of a typical Spring Boot project structure and key components:

Folder Structure

src/
├── main/
│   ├── java/                  # Java source files
│   │   └── com.example.demo/  # Package for Java classes
│   │       ├── controller/    # Controllers for handling web requests
│   │       ├── model/         # Model classes for data entities
│   │       ├── repository/    # Data access layer interfaces
│   │       └── service/       # Service classes for business logic
│   ├── resources/             # Resource files
│   │   ├── static/            # Static resources (e.g., CSS, JavaScript)
│   │   ├── templates/         # HTML templates (e.g., Thymeleaf, FreeMarker)
│   │   └── application.properties  # Application configuration properties
├── test/                      # Test source files
└── pom.xml                    # Maven build configuration

Application Class

package com.example.demo;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class DemoApplication {
   public static void main(String[] args) {
       SpringApplication.run(DemoApplication.class, args);
   }
}

Controller

package com.example.demo.controller;

import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class HelloController {

   @GetMapping("/hello")
   public String hello() {
       return "Hello, World!";
   }
}

Model

package com.example.demo.model;

import javax.persistence.Entity;
import javax.persistence.Id;

@Entity
public class User {

   @Id
   private Long id;

   // Getters and setters
}

Repository

package com.example.demo.repository;

import com.example.demo.model.User;
import org.springframework.data.jpa.repository.JpaRepository;

public interface UserRepository extends JpaRepository {
}

Service

package com.example.demo.service;

import com.example.demo.model.User;
import com.example.demo.repository.UserRepository;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;

@Service
public class UserService {

   @Autowired
   private UserRepository userRepository;

   public User getUserById(Long id) {
       return userRepository.findById(id).orElse(null);
   }
}

Spring Boot Database Connection

In a Spring Boot application, you can configure the database connection using properties in the application.properties or application.yml file. Here's how you can set up a MySQL database connection:

1. Add MySQL Dependency

Include the MySQL connector dependency in your pom.xml if you're using Maven:

<dependency>
   <groupId>mysql</groupId>
   <artifactId>mysql-connector-java</artifactId>
</dependency>

Or if you're using Gradle, add the dependency in your build.gradle:

implementation 'mysql:mysql-connector-java'

2. Configure Database Connection

In your application.properties or application.yml, specify the database connection properties:

Using application.properties:

spring.datasource.url=jdbc:mysql://localhost:3306/your_database_name
spring.datasource.username=your_database_username
spring.datasource.password=your_database_password
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver

Using application.yml:

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/your_database_name
    username: your_database_username
    password: your_database_password
    driver-class-name: com.mysql.cj.jdbc.Driver

Note: Replace your_database_name, your_database_username, and your_database_password with your actual database name, username, and password.

3. Enable JPA and Specify Entity Scan

In your main application class, enable JPA and specify the package to scan for entities:

package com.example.demo;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.data.jpa.repository.config.EnableJpaRepositories;

@SpringBootApplication
@EnableJpaRepositories(basePackages = "com.example.demo.repository")
public class DemoApplication {
   public static void main(String[] args) {
       SpringApplication.run(DemoApplication.class, args);
   }
}

4. Create Entity and Repository

Create a JPA entity representing your database table and a repository interface for database operations:

Entity:

package com.example.demo.model;

import javax.persistence.Entity;
import javax.persistence.GeneratedValue;
import javax.persistence.GenerationType;
import javax.persistence.Id;

@Entity
public class User {
   @Id
   @GeneratedValue(strategy = GenerationType.IDENTITY)
   private Long id;
   private String name;
   private String email;

   // Getters and setters
}

Repository:

package com.example.demo.repository;

import com.example.demo.model.User;
import org.springframework.data.jpa.repository.JpaRepository;

public interface UserRepository extends JpaRepository<User, Long> {
}

5. Use Repository in Service or Controller

You can now use the UserRepository in your service or controller to perform database operations:

package com.example.demo.service;

import com.example.demo.model.User;
import com.example.demo.repository.UserRepository;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;

@Service
public class UserService {

   @Autowired
   private UserRepository userRepository;

   public User getUserById(Long id) {
       return userRepository.findById(id).orElse(null);
   }
}

This is a basic setup for connecting a Spring Boot application to a MySQL database using Spring Data JPA.
